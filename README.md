# Javi — Local Voice AI Orchestrator

A warm, "Javi"-style voice interface for a **100% local** AI assistant: a
reactive particle orb, text + voice input, hands-free listening, and spoken
replies — wired to a local model via [Ollama](https://ollama.com). Everything
runs on your machine (see [Privacy](#privacy)).

The UI is implemented from the **Javi** Claude Design (four warm themes, serif
display type). The underlying brain/audio logic is unchanged from the original
prototype, kept in `legacy/jarvis.html` for reference.

## Requirements

- **Node ≥ 20** (the toolchain is Vite 8 / Vitest 4). An `.nvmrc` pins the
  version — run `nvm use` in this folder. The nvm default on this machine is an
  old v14, so don't forget to switch.
- **Chrome / Chromium** — speech recognition is Chromium-only.

## Quick start

### 1. Ollama (the brain) with CORS open

The browser calls Ollama directly, so the container must allow cross-origin
requests via `OLLAMA_ORIGINS`. `OLLAMA_KEEP_ALIVE=-1` keeps the model resident
in VRAM so the first prompt isn't slow.

```bash
docker rm -f ollama
docker run -d --device /dev/kfd --device /dev/dri \
  -v ollama:/root/.ollama -p 11434:11434 \
  -e OLLAMA_ORIGINS="*" \
  -e OLLAMA_KEEP_ALIVE=-1 \
  --name ollama ollama/ollama:rocm

docker exec -it ollama ollama pull gemma3:12b
```

### 2. Frontend

```bash
nvm use          # Node 20+ (see .nvmrc)
npm install
npm run dev      # serves http://localhost:8080
```

Open **http://localhost:8080** in **Chrome/Chromium**. The mic and Web Speech
APIs require a secure origin — Vite's `localhost` dev server qualifies, so don't
open `index.html` via `file://`.

### Build

```bash
npm run build    # outputs to dist/
npm run preview
npm test         # vitest (state machine)
```

## Using it

- **Themes** — the four swatches (top-right) switch between Coral, Salvia,
  Terracota and Azul Noche. Your choice is remembered (localStorage).
- **Hands-free listening** — tap the mic (or the **Escuchando** segment) to start
  a continuous listening session. Javi transcribes, answers, and keeps listening
  — no need to press the button each turn. Tap again (or **En reposo**) to stop.
- **Typing** — just type in the bar and hit Enter / the send button.
- The center caption shows the live transcript while you speak and Javi's reply
  as it streams.

## Project structure

```
jarvis/                      # folder name kept; product is "Javi"
├── index.html               # markup only; loads /src/main.js as a module
├── config.js                # Ollama host/model, default theme, voice, prompt, flags
├── src/
│   ├── main.js              # composition root — wires every module together
│   ├── state/               # central state machine (idle/listening/thinking/speaking)
│   ├── orb/                 # DOM particle orb (4 shapes, seeded layouts, reactive)
│   ├── audio/               # mic-meter (FFT) · stt (continuous) · tts (synthesis)
│   ├── brain/               # Ollama streaming client + conversation history
│   └── ui/
│       ├── themes.js         # 4 themes → CSS custom properties + swatch picker
│       ├── state-presentation.js  # state → Spanish label / caption / orb shape
│       ├── hud.js · controls.js · styles.css
└── legacy/jarvis.html       # original single-file prototype (reference only)
```

**Design notes**
- *State machine + pub/sub*: `src/state/machine.js` is the single source of
  truth; the orb, HUD and controls subscribe. No module imports another — they
  meet only in `main.js`. This is also the seam the Phase 4 orchestrator uses.
- *Particle orb*: each state maps to a shape (orbit / scatter / spiral / ring)
  generated by a seeded RNG so layouts are deterministic. A single `amplitude`
  value (0..1) drives `--orb-energy` for reactivity — fed by mic FFT while you
  talk, TTS boundary pulses while Javi talks.
- *Settings moved to `config.js`*: the design has no settings panel, so model /
  voice / host are configured there. The app auto-detects the model
  (`gemma3:12b` if present) and a Spanish voice at boot.

## Roadmap

- **Phase 2 — better voice (highest impact):** swap browser TTS for a local
  [Piper](https://github.com/rhasspy/piper) microservice; its audio *is*
  FFT-capturable, so the orb can react with real FFT while Javi speaks.
  (Flip `config.features.usePiper`.)
- **Phase 3 — private STT:** swap `webkitSpeechRecognition` for a local
  [whisper.cpp](https://github.com/ggerganov/whisper.cpp) microservice.
  (Flip `config.features.useWhisper`.)
- **Phase 4 — orchestration:** let the local model route heavy work to **n8n**
  (webhooks) and **Claude Code** (deep reasoning) via tool-calling. The seam is
  stubbed in `src/brain/ollama.js` (`registerTool` + `tools`); nothing is wired
  yet.

## Privacy

Local-first. The only thing that currently leaves your machine:

- **`webkitSpeechRecognition`** sends microphone audio to Google for
  transcription. Phase 3 removes this. Use text input to stay fully local.

Everything else — the model, TTS, the UI — runs locally.

## Hardware notes

Tuned for: AMD RX 9070 XT (16GB VRAM, RDNA4), Ryzen 5 2600, 32GB RAM, Ubuntu.
Driver model is `gemma3:12b` (Q4_K_M, ~8GB — fits entirely in VRAM). **Models
must fit fully in 16GB of VRAM**; spilling to RAM tanks throughput (~14× slower)
on this CPU, so avoid dense 24B+ models as the primary driver.
